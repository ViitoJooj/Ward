# Security Policy

Ward takes security seriously.

## Supported versions

At this stage, security fixes are applied to the latest code on the default branch.

## Reporting a vulnerability

If you find a potential vulnerability, please report it responsibly:

1. Do not publish exploit details before maintainers have time to fix.
2. Open a private report if possible, or contact maintainers directly.
3. Include clear reproduction steps, impact, and affected files/areas.

## Disclosure process

1. Maintainers validate the issue.
2. A fix is prepared and reviewed.
3. A patch is released.
4. Public disclosure can happen after users have a reasonable update window.

## Security expectations for contributors

All contributors (including AI-assisted contributions) must:

1. Avoid introducing known insecure patterns.
2. Validate auth/authorization logic carefully.
3. Handle secrets and tokens safely.
4. Preserve secure defaults and explicit error handling.
5. Flag any security-sensitive change clearly in the PR.

## Ethical use statement

Ward is free for personal, startup, and enterprise use.
Do not use this project to cause harm to people, systems, or communities.
