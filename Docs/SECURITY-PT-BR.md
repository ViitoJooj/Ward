# Politica de Seguranca

O Ward trata seguranca como prioridade.

## Versoes suportadas

Neste momento, correcoes de seguranca sao aplicadas na versao mais recente da branch principal.

## Como reportar uma vulnerabilidade

Se voce encontrar uma vulnerabilidade potencial, reporte de forma responsavel:

1. Nao publique detalhes de exploracao antes de existir correcao.
2. Abra um reporte privado quando possivel, ou entre em contato direto com os mantenedores.
3. Envie passos de reproducao, impacto e areas afetadas.

## Processo de divulgacao

1. Mantenedores validam o problema.
2. A correcao e preparada e revisada.
3. O patch e publicado.
4. A divulgacao publica pode ocorrer apos janela razoavel de atualizacao.

## Expectativas de seguranca para contribuidores

Todo contribuidor (inclusive com apoio de IA) deve:

1. Evitar padroes inseguros conhecidos.
2. Revisar com cuidado logica de autenticacao/autorizacao.
3. Tratar segredos e tokens com seguranca.
4. Manter defaults seguros e erros explicitos.
5. Sinalizar no PR quando a mudanca for sensivel para seguranca.

## Declaracao de uso etico

Ward e gratuito para uso pessoal, startups e empresas.
Nao utilize este projeto para causar dano a pessoas, sistemas ou comunidades.
